Automatic Differentiation

An interactive exploration of forward & reverse mode AD — and why it powers modern deep learning

The gradient problem

Training a neural network requires computing the gradient of the loss with respect to every parameter — potentially billions of them. Three strategies exist, and they differ wildly in cost.

🔢 Numerical Differentiation

O(n) evals

For each parameter θi, perturb it by ε and measure Δf. Needs n extra forward passes. Impractical for large models; also inexact due to floating-point rounding.

∂f/∂xᵢ ≈ (f(x+εeᵢ)−f(x)) / ε

✏️ Symbolic Differentiation

Expression blow-up

Applies calculus rules to produce an exact symbolic formula. The resulting expression can be exponentially larger than the original — a problem called expression swell.

d/dx(x·sin(x)) = sin(x) + x·cos(x)

⚡ Automatic Differentiation

O(1) overhead

Evaluates derivatives alongside values using the chain rule on a computation graph. All n gradients cost only ~2–4× a single forward pass — regardless of n.

∂L/∂θᵢ for all i in one backward pass

The chain rule — the key idea

Any computation can be broken into primitive operations (+, ×, sin, …). Each primitive has a known local derivative. The chain rule lets us compose them:

∂z/∂x = (∂z/∂v)·(∂v/∂x) + (∂z/∂w)·(∂w/∂x)

Forward mode propagates tangents (∂v/∂x) left-to-right through the graph — best when inputs are few.

Reverse mode propagates adjoints (∂L/∂v) right-to-left — best when outputs are few (e.g., a scalar loss). This is backpropagation.

In deep learning: one scalar loss, millions of params → reverse mode wins.

Forward Mode: Dual Numbers

Forward mode AD augments every real number with a tangent component, tracking how the value changes with respect to one chosen input variable. A dual number is written as:

x + ε·ẋ    where ε² = 0

At any leaf node, we set ẋ = 1 for the variable we're differentiating with respect to (the "active variable"), and ẋ = 0 for all others. Each arithmetic operation propagates both the value and its derivative simultaneously — no separate backward pass needed.

Forward mode is most efficient when the number of inputs is small (e.g., computing a full Jacobian column-by-column).

Dual arithmetic rules

(a + aε) + (b + bε) = (a+b) + (a+b)ε
(a + aε) · (b + bε) = ab + (ab+ab)ε
sin(a + aε) = sin(a) + a·cos(a)·ε
exp(a + aε) = eᵃ + a·eᵃ·ε

The tangent component (coefficient of ε) exactly tracks the derivative — no approximation, no extra function evaluations.

Since ε² = 0, all higher-order terms vanish automatically. This makes dual numbers both exact and cheap.

Each rule is just the chain rule applied to one primitive operation.

Why does the learning rate matter so much?

Too large (η ≫ optimal)

Overshoots the minimum on each step — loss may oscillate or diverge entirely.

Just right (η ≈ optimal)

Converges smoothly and quickly to the minimum.

Too small (η ≪ optimal)

Converges, but takes many more iterations — impractical for large models.

Adam adapts the learning rate per-parameter using estimates of the first and second moments of the gradients — making it much more robust to the choice of η.
