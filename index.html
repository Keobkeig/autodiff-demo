Why Automatic Differentiation?

The gradient problem

Training a neural network requires the gradient of the loss with respect to every parameter — potentially billions of them. Three approaches exist, but only one scales.

O(n) evaluations

🔢 Numerical

Perturb each θᵢ by ε, measure Δf. Needs n extra forward passes. Impractical and inexact due to floating-point error.

∂f/∂xᵢ ≈ (f(x+εeᵢ)−f(x)) / ε
Expression swell

✏️ Symbolic

Applies calculus rules to produce an exact formula — but the resulting expression can grow exponentially larger than the original.

d/dx(x·sin x) = sin x + x·cos x
~2× forward cost

⚡ Automatic (AD)

Evaluates derivatives alongside values using the chain rule on a computation graph. All n gradients in one backward pass.

∂L/∂θᵢ for all i — one pass

The chain rule — the key idea

Break any computation into primitive operations (+, ×, sin, …). Each primitive has a known local derivative. The chain rule composes them:

∂z/∂x = (∂z/∂v)·(∂v/∂x) + (∂z/∂w)·(∂w/∂x)

Forward mode propagates tangents left-to-right — efficient when inputs are few.

Reverse mode propagates adjoints right-to-left — efficient when outputs are few (one scalar loss). This is backpropagation.

💡In deep learning: one scalar loss, millions of params → reverse mode always wins.

Why it's O(1) overhead

For a function f : ℝⁿ → ℝ, the backward pass visits each node exactly once, accumulating gradient contributions. Total work is proportional to the circuit size — not n².

This was the key insight behind the 1986 backprop paper and every modern deep-learning framework (PyTorch, JAX, TensorFlow) today.

📌The green forward pass + red backward pass above is exactly how PyTorch's autograd works under the hood.

Supported ops in this demo: + − × ÷ ^ sin cos exp ln tanh

Forward Mode: Dual Numbers

Forward mode AD augments every number with a tangent component, tracking how the value changes with respect to one chosen input. A dual number is:

a + ε·ȧ     where ε² = 0

Set ȧ = 1 for the active variable, ȧ = 0 for all others. Every arithmetic rule naturally propagates both value and derivative — no separate backward pass needed.

💡Forward mode is most efficient when the number of inputs is small (computing a Jacobian column-by-column).

Dual arithmetic rules

Each rule is the chain rule applied to one primitive. Since ε² = 0, all higher-order terms vanish automatically — making dual arithmetic both exact and efficient.

(a + ȧε) + (b + ḃε) = (a+b) + (ȧ+ḃ)ε
(a + ȧε) · (b + ḃε) = ab + (aḃ+ȧb)ε
sin(a + ȧε) = sin a + ȧ·cos a · ε
exp(a + ȧε) = eᵃ + ȧ·eᵃ · ε

The tangent component (coefficient of ε) exactly tracks the derivative at each step — no approximation, no extra function evaluations.

Gradient Descent in Action

Once we can compute gradients cheaply via AD, we minimize a loss function by iteratively stepping opposite the gradient. Here we train a linear model y = wx + b on synthetic data using MSE loss:

L(w, b) = (1/n) Σ (w·xᵢ + b − yᵢ)²

The gradient ∂L/∂w and ∂L/∂b are computed analytically here — the Gradient Descent tab illustrates what AD enables, while the Computational Graph tab shows how it computes those gradients.

Why the learning rate matters

Too large — diverges

May look okay for a few steps, then overshoots and explodes as gradients grow. Watch for late spikes in the loss curve. Try η = 0.5 above.

Just right — converges

Descends smoothly to the minimum. The loss curve falls cleanly. Try η ≈ 0.05.

Too small — crawls

Converges, but takes far more iterations. Impractical for large models. Try η = 0.001.

⚡Adam adapts the learning rate per-parameter using first and second gradient moment estimates — dramatically more robust to the choice of η.
